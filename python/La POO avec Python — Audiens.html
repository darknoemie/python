Introduction à la POO

Le lien vers le projet dans gitlab : https://gitlab.audiens.fr/python/cours/tree/master/poo

Les classes

une classe = propriétés + méthodes

  • propriétés = équivalent aux variables classiques
  • méthodes sont des fonctions
import sys

class Figure:

    # Variable de classe
    compteur = 0
    # Constructeur
    def __init__(self, nom):
        # variable d'instance
        self.nom = nom
        self.couleur = "rouge"
        Figure.compteur += 1

    def affiche(self):
        sys.stdout.write("figure nom="+self.nom)

Visibilité des membres de la classe

Encapsulation des données

On protège les données en les encapsulant dans des classes.

  • Python n'a pas de mot clé private
  • Pour spécifier un attribut privé, il faut mettre deux caractères __ au début du nom.

Avantages de l'encapsulation :

  • en cas de problème de manipulation de la donnée, on réduit la zone d'investigation.

Encapsulation forte

  • les données membres sont privées et jamais "public".
  • les fonctions membres (méthodes) sont "public" et quelques fois privées.

on accède alors aux données par des méthodes (getter / setter) ou par des propriétés

Notion de signature de fonction

La signature d'une fonction (méthode) est :

  • le nom de la méthode
  • la liste ordonnée des arguments passés


Surcharge de fonctions

La surcharge d'une méthode consiste à écrire une méthode qui porte le même nom , mais avec des arguments différents.

  • Dans python la surcharge de méthode se fait via l'héritage.
  • La surcharge de méthode n'est pas possible à l'intérieur d'une même classe

Variables de classe

Les variables de classe :

  • ont une durée de vie indépendante de la durée de vie des instances.
  • sont partagées par toutes les instances.
  • sont définies dans le namespace de la classe

    # Variable de classe
    vc = 'variable de classe'

  • on y accède par la syntaxe suivante :

Méthodes de classe

Les méthodes de classe peuvent être invoquées sans créer d'instance. Exemple : 

import sys

class Figure:

    # Variable de classe
    compteur = 0

    # Methode de classe , le premier argument doit être cls
    # cls identifie la classe
    @classmethod
    def cm_getCompteur(cls):
        return cls.compteur

    # Deuxieme methode  de classe
    @staticmethod
    def sm_getCompteur():
        return Figure.compteur

    # Constructeur
    def __init__(self, nom):
   ....
   ....

L'utilisation se fait de la manière suivante : 

print("""Utilisation des méthodes de classe""")
x=Figure.Figure.cm_getCompteur()
print(x)
print("Utilisation d'une methode static")
y=Figure.Figure.sm_getCompteur()
print(y)

Notion de constructeur

  • le constructeur est une méthode de la classe
  • il porte le même nom que la classe
  • n'a pas de type
  • il peut être surchargé
  • il est appelé automatiquement lorsque la variable entre dans la portée du programme.
import sys

class Figure:
    ....
    ....
    # Constructeur
    def __init__(self, nom):
        """ variable d'instance """
        self.nom = nom
        self.couleur = "rouge"
        Figure.compteur += 1

Héritage

Définition :

  • Il s'agit de créer une classe à partir d'une autre classe.
  • La classe dérivée récupère les propriétés et les méthodes.
  • créer la classe Figures avec les propriétés nom et couleur et la méthode affiche();
  • ajouter un constructeur par défaut pour initialiser la couleur.
  • ajouter un constructeur pour initialiser les données membres.
  • créez la classe Cercle dérivée de Figure
import sys
from dessin.Figure import Figure

""" La classe Cercle hérite de figure """
class Cercle(Figure):

    def __init__(self,nom):
        # Appel du constructeur de la classe de base
        super().__init__(nom)
        self.rayon = 10
        self.x = 150
        self.y = 150

    def affiche(self):
        ....
        ....

Puis instanciez des objets dans le programme principal

Dérivation

la classe dérivée doit faire appel à au moins un constructeur de la classe de base. 

class Cercle(Figure.Figure):

    def __init__(self,nom):
        # Appel du constructeur de la classe de base
        super().__init__(nom)

Surcharge des méthodes dérivées. 

Appel d'une méthode de la classe de base : super.affiche();

Polymorphisme

En programmation objet on peut manipuler une liste d'objet hétérogènes appartenant tous à la même hiérarchie de classes.

On ajoute les instructions suivantes pour créer une liste polymorphe 

   ....
   figure = Figure.Figure('figure1')
   cercle = Cercle.Cercle("cercle1");
   
   # Illustration du polymorphisme
   print("-- Illustration du polymorphisme --")
   liste=[figure, cercle]
   for f in liste:
     f.affiche()
     print()
   ...

Le résultat est 

-- Illustration du polymorphisme --
figure nom=figure1()
cercle1,rayon=10,x=150,y=150()
